# CityReport.io Municipal Pilot Overview

## Pilot Scope
- Domains: `potholes`, `water_drain_issues`
- Streetlights: utility-information layer only (not municipal accountability domain)
- Aggregation: proximity-based for both pilot domains

## Pilot Duration
- Recommended: 30-60 days

## Intake Sources
- Citizens
- Municipal staff
- Source tracking in reports:
  - `citizen`
  - `municipal_staff`
  - `system`

## What City Staff Can Do
- View reports by status (open/closed/all)
- Search/filter incidents by date and query
- Mark incidents fixed with notes
- Export summary/detail CSV

## Data Ownership and Access
- Municipal pilot data remains in tenant-scoped Supabase project
- Public/internal visibility follows Model C controls
- Internal controls retain detailed operational context

## Success Criteria
- Reduced average time-to-fix
- Lower reopen rate
- Identification of chronic locations
- Improved clustering/triage efficiency

## Expected Pilot Outcome
- Demonstrate measurable operational accountability
- Produce defensible metrics for procurement review

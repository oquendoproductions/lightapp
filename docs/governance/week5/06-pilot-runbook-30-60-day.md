# Week 5 — Pilot Runbook (30–60 Days)

## Day 0 Setup
- Confirm pilot users and access levels.
- Confirm pilot domains enabled: `potholes`, `street_signs`.
- Confirm streetlights are excluded from pilot SLA/accountability narrative.

## Daily Ops Checklist
- Check Open Reports queue consistency.
- Spot-check fixed vs listed incident consistency.
- Check moderation flag count and severity trend.
- Validate map + modal core flows are responsive.

## Weekly Ops Checklist
- Export summary CSV and detail CSV for pilot window.
- Review KPI scorecard fields.
- Review reopen/chronic outliers.
- Capture top 3 user friction points.

## Incident Response Rules
- P0 (data correctness/security): patch immediately, notify sponsor.
- P1 (workflow blocking): patch within 24h.
- P2 (usability clarity): batch into weekly polish.

## Change Control During Pilot
- Allowed:
  - Copy/label clarity.
  - Layout consistency.
  - Bug fixes and data hygiene fixes.
- Not allowed:
  - New domains.
  - New lifecycle states.
  - Export schema/metric formula changes.

## Day 30 Review Package
- KPI trend summary.
- Reliability and risk summary.
- Stakeholder feedback digest.
- Recommendation: extend, convert, or adjust.

## Day 60 Decision Package (if extended)
- Procurement fit summary.
- Proposed pricing/scope tiers.
- Phase-2 implementation proposal.

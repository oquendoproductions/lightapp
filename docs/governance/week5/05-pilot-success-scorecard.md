# Week 5 — Pilot Success Scorecard

## Cadence
- Reporting period: weekly.
- Audience: municipal sponsor + internal ops lead.

## Core KPI Block
- Total reports submitted (weekly).
- Incidents opened (weekly).
- Incidents fixed (weekly).
- Avg time-to-fix (weekly + rolling).
- Reopen rate (% reopened / fixed).
- Chronic incidents count (`reopen_count >= 2 within 90 days`).

## Operational Quality Block
- Export success rate (% successful exports).
- Data integrity checks passed (yes/no).
- Open Reports stale/deleted asset anomalies (count).
- Moderation flags open (count by severity).

## User Experience Block
- Admin-reported blockers (count).
- Time-to-triage median (internal estimate).
- Confusing labels/flows requiring clarification (count).

## Risk Block
- Permission/access incidents.
- Console/system noise observed in pilot workflows.
- Domain ownership confusion raised by stakeholders.

## Decision Thresholds
- Strong pilot
  - Stable KPIs for 3+ consecutive weeks.
  - No unresolved high-severity data integrity issues.
  - Sponsor requests continuation/expansion discussion.

- At-risk pilot
  - Repeated metric confusion.
  - Data hygiene regressions.
  - Workflow friction preventing routine use.
